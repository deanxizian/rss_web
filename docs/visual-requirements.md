# 视觉要求

本文档记录 RSS AI Reader 当前约定的视觉与布局要求。后续调整界面时，应优先保持这些规则一致。

## 整体风格

- 产品定位是个人使用的 RSS 阅读与 AI 内容处理工具，界面应偏向工作台和阅读器，而不是营销页。
- 首屏直接进入可用界面，不做落地页、宣传区或大幅装饰性视觉。
- 整体采用深色工作台风格，视觉重点放在订阅、文章、正文和处理结果上。
- 配色保持克制，主操作使用绿色强调色，普通内容使用深色面板、细边框和低饱和文字层级。
- 模块边角保持 8px 或更小，不使用夸张圆角、渐变装饰、漂浮装饰元素。
- 文案应简洁、专业，避免说明书式长句。界面文字只描述当前状态和下一步动作。

## 顶部区域

- 顶部栏在桌面宽度下固定在页面顶部，保持轻量、紧凑。
- 品牌文案为 `RSS AI Reader`，副标题为 `RSS 阅读与 AI 内容处理`。
- RSS 输入框、读取、导入、导出位于顶部主操作区。
- 按钮顺序固定为：`读取`、`导入`、`导出`。
- `读取` 始终紧挨 RSS 输入框。
- 多列模式下三个按钮宽度一致。
- 单列模式下，`读取` 保持在输入框右侧；`导入` 和 `导出` 位于下一行，宽度一致并撑满模块。
- RSS 输入框宽度应保持稳定，不因按钮状态或文字变化产生明显跳动。
- 支持将 OPML 文件拖到 RSS 输入区域，拖拽状态只做轻量边框高亮。

## 模块结构

页面由以下大模块组成：

- `偏好`：模型、翻译、朗读和访问口令设置。
- `RSS 订阅`：本地保存的 RSS 源，支持清空、导入、导出。
- `文章`：当前 RSS 源下的文章列表。
- `原文`：当前文章正文，使用 Markdown 风格展示。
- `工作台`：总结、翻译、全文朗读三个结果模块。

每个大模块应保持统一的面板样式：

- 顶部为模块标题、说明和右上角图标或操作按钮。
- 内容区留出稳定内边距。
- 面板之间使用一致间距。
- 不在模块中再嵌套大卡片；只有结果窗口、列表项和必要控件使用内部框。

## 响应式布局

页面按可用宽度切换列数：

- 单列：所有模块纵向排列。
- 2 列及以上：最左列固定宽度，其他列等宽。
- 2 列：`偏好`、`RSS 订阅`、`文章` 在第一列；`原文`、`工作台` 在第二列。
- 3 列：`偏好`、`RSS 订阅`、`文章` 在第一列；`原文` 在第二列；`工作台` 在第三列。
- 4 列：`偏好`、`RSS 订阅` 在第一列；`文章` 在第二列；`原文` 在第三列；`工作台` 在第四列。
- 正常桌面全屏应优先呈现 4 列布局。
- 4 列下第一列保持固定宽度，其他三列宽度一致。
- 页面最大宽度保持约束，避免超宽屏上内容过度拉伸。

## 高度与对齐

- 多列模式下，`原文` 是主要高度基准，尽量撑满可视高度，但高度略短于整屏，避免视觉过满。
- 多列模式下，`工作台` 模块高度必须始终与 `原文` 模块一致。
- 多列模式下，同一行的大模块顶部应对齐；需要跨行排列的模块，应通过底部对齐保持整体秩序。
- 单列模式下不强制跨模块底部对齐，按自然纵向顺序排列即可。
- 多列模式下，最左列底部必须与右侧最底部模块底部对齐。
- 多列模式下，最左列里的可见内容模块应平分列高。
- `偏好` 折叠时，`偏好` 只保留折叠后的自然高度；最左列剩余模块平分剩余高度。
- 2 列模式下，最左列底部必须对齐 `工作台` 底部，而不是只对齐 `原文` 底部。
- 2 列模式下，`原文` 和 `工作台` 在右列上下排列，两者高度一致。
- 2 列模式下，`RSS 订阅` 和 `文章` 之间不能出现明显空白，也不能互相挤压。
- 3 列模式下，`原文` 和 `工作台` 高度一致；第一列底部对齐二者底部。
- 4 列模式下，`文章`、`原文`、`工作台` 高度一致，三列顶部和底部都应对齐。
- 4 列模式下，第一列固定宽度，`文章`、`原文`、`工作台` 三列等宽；第一列底部也应与这些模块底部对齐。
- 模块内容较少时保持整齐对齐；内容较多时优先保证模块边界稳定，再通过内部滚动承载内容。

### 对齐状态矩阵

| 页面状态 | 对齐要求 |
| --- | --- |
| 单列 | 所有模块自然纵向排列，不要求底部对齐；模块间距保持一致。 |
| 2 列，偏好展开 | 第一列包含 `偏好`、`RSS 订阅`、`文章`，三者平分第一列高度；第一列底部对齐右列 `工作台` 底部；`原文` 与 `工作台` 高度一致。 |
| 2 列，偏好折叠 | `偏好` 保持折叠高度；`RSS 订阅` 与 `文章` 平分第一列剩余高度；第一列底部对齐右列 `工作台` 底部；`原文` 与 `工作台` 高度一致。 |
| 3 列，偏好展开 | 第一列包含 `偏好`、`RSS 订阅`、`文章`，三者平分第一列高度；第一列底部与 `原文`、`工作台` 底部对齐。 |
| 3 列，偏好折叠 | `偏好` 保持折叠高度；`RSS 订阅` 与 `文章` 平分第一列剩余高度；第一列底部与 `原文`、`工作台` 底部对齐。 |
| 4 列，偏好展开 | 第一列包含 `偏好`、`RSS 订阅`，二者平分第一列高度；第一列、`文章`、`原文`、`工作台` 底部对齐。 |
| 4 列，偏好折叠 | `偏好` 保持折叠高度；`RSS 订阅` 填满第一列剩余高度；第一列、`文章`、`原文`、`工作台` 底部对齐。 |
| 4 列空状态 | `读取 RSS 后显示文章`、`选择一篇文章查看正文`、`选择一篇文章后开始处理` 三个提示的视觉中心线必须一致。 |
| RSS 源很多 | `RSS 订阅` 列表内部滚动；不撑破第一列，不影响最左列与右侧模块的底部对齐。 |
| 文章很多 | `文章` 列表内部滚动；外层模块高度不被列表撑高。 |
| 正文很长 | `原文` 正文区域内部滚动；`原文` 模块高度不变。 |
| 总结、翻译结果很长 | 对应结果窗口内部滚动；`工作台` 模块高度不变。 |
| 窗口高度较低 | 允许页面整体滚动；必要时允许 `偏好` 内容区内部滚动，但不能破坏最左列模块平分和底部对齐。 |

## 滚动行为

- 页面可以整体滚动，尤其是在窗口高度较小时。
- 不应让第一列出现“偏好”和“RSS 订阅”共用的内部滚动条；这种情况应交给整页滚动或具体列表滚动处理。
- `偏好` 展开时，在常规桌面高度下尽量不要出现内部滚动。
- 如果窗口高度不足，`偏好` 可以在内容区内部滚动，但仍需保持最左列的高度分配和底部对齐。
- RSS 源数量很多时，`RSS 订阅` 列表内部滚动，不能撑破模块。
- 文章很多时，`文章` 列表内部滚动，不能撑破模块。
- `原文` 正文区域内部滚动，保持标题和外部模块稳定。
- `总结`、`翻译`、`全文朗读` 的结果窗口内部滚动，保持工作台整体结构稳定。

## 控件要求

- 主要操作按钮使用绿色主按钮样式。
- `读取`、`导入`、`导出` 都是主按钮。
- `生成总结`、`翻译全文`、`生成音频` 都是主按钮，并分别放在对应模块区域内。
- 大模块右上角应使用图标或明确的操作按钮，避免冗长文字。
- 清空订阅使用图标按钮。
- 模型、目标语言、朗读语言、朗读音色使用下拉框。
- 朗读速度使用滑杆，并显示当前倍率。
- 目标语言和朗读语言之间不需要分割线。
- 可交互元素高度保持一致，按钮和输入框在同一行时应垂直对齐。

## 文本与溢出

- 所有长链接、标题、作者、RSS 源地址和 Markdown 链接都必须允许换行，不能超出模块。
- 按钮文字不能换行、裁切或挤压图标。
- 模块说明文字保持短句，避免占用过多高度。
- 空状态文案应描述下一步动作，例如读取 RSS、选择文章、开始处理。
- 状态栏用于显示当前处理状态、错误和成功消息，文字应短而明确。

## 原文与 Markdown

- 原文模块从工作台中独立出来，作为单独大模块展示。
- 原文内容尽量显示为 Markdown 风格，包括段落、标题、列表、引用、代码块和链接。
- 原文模块图标使用文档语义图标。
- `查看原文` 链接应清晰但不抢占主视觉。

## 工作台

- 工作台固定包含三个不折叠模块：`文章总结`、`全文翻译`、`全文朗读`。
- 三个模块在工作台中始终可见，不隐藏在标签页或折叠区里。
- 每个模块的操作按钮放在对应模块右上角。
- 总结、翻译和音频结果互相独立，不共享一个结果区域。
- 音频支持分段生成和分段播放，分段信息应清楚但不喧宾夺主。

## 验证要求

视觉相关改动完成后，应至少检查：

- 单列、2 列、3 列、4 列布局是否合理。
- 4 列桌面宽度下是否没有横向溢出。
- 多列模式下最左列模块是否按展开或折叠状态正确分配高度。
- 多列模式下最左列底部是否对齐右侧最底部模块。
- 2 列模式下 `原文` 与 `工作台` 是否同高，且第一列底部是否对齐 `工作台` 底部。
- 空状态、长标题、长链接、大量 RSS 源、大量文章时模块是否整齐。
- 构建命令 `next build` 是否通过。
